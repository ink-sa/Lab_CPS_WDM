Проект разработки специализированного языка программирования для сторожевых механизмов.

1. Компилятор с разрабатываемого языка compil.py находится в папке Py_WDM. Здесь же находится эмулятор системы -  system_emulation.py
2. Для того, чтобы скомпилировать промежуточный код необходимо разместить файл с текстом исходного кода в папке Py_WDM. 
После этого в окружении папаки Py_WDM запустить командную строку и набрать в ней: python compil.py имя_вашего_файла_с_расширением и нажать Enter. 
Если файл будет найден и успешно скомпилируется, в папке User обновиться файл main.c(компилятор генерирует текст на Си, записываемый в файл main.c, лежащий в папке User).
3. В качестве конечной платформы, для которой пишется программа выступает отладочная плата STM32F4Discovery.
4. Для компиляции конечной программы необходимо открыть проект в keil uvision версии не ниже 4 и запустить компиляцию.
5. Полученную программу можно зашить в STM32F4Discovery либо через keil, либо использовать иные утилиты для прошивки и 
использовать сгенерированный файл Lab_CPS_WDM.hex из папки Objects.
6. После запуска программы на микроконтроллере, можно запустить эмулятор системы  system_emulation.py. 
7.В качестве тестовой программы на специализированном языке, которой соответствует эмулятро необходимо использовать следующую последовательность:

STATEID = 1;
TIMEOUT = 1400;
NEXTSTATE: 2;

STATEID = 2;
TIMEOUT = 1500;
PARAM 1: = 1;
NEXTSTATE: 2, 3;

STATEID = 3;
TIMEOUT = 1500;
PARAM 2: > 5;
NEXTSTATE: 1, 2;

8. Необходимо также предвварительно настроить номер COM порта, с которым будет работать эмулятор (это делается 
в файле system_emulation.py - по умолчанию стоит COM13).
9. Для подключения к плате STM32F4Discovery необходимо использовать переходник USB-UART, который следует подключить к
выводам PB6(TX) и PB7(TX).UART работает на скорости 115200/8 бит/1стоп-бит/без аппаратного управления.
10. После прошивки платы тестовой программой и запуска программы-имитатора системы на плате при нормальной работе программы должны попеременно загораться зеленый и синий диоды.  
Если происходит сбой, тестовая программа будет перезагружена, о чем будет свидетельствовать зажигание красного диода на плате.
